What is TAC?

Three Address Code (TAC) is an intermediate representation used in compilers. Each instruction has at most three operands and follows a simple format:

result = operand1 operator operand2
Note: TAC breaks down complex expressions into simpler ones, making it easier for compilers to optimize and generate machine code.

Basic Operations

Common TAC operations include:

  • Assignment: x = y
  • Arithmetic: t1 = a + b
  • Comparison: t2 = x > y
  • Control flow: ifFalse t1 goto 200

Common Pitfalls

Warning: Always check your goto addresses!

Common mistakes to avoid:

  • Incorrect goto addresses
  • Missing temporary variables
  • Improper control flow

Nested Structures

Input C code:

Tip: Nested structures require careful management of goto addresses to maintain proper control flow.

TAC Generation Process

Step 1: Initialization

First, we initialize variables and set up the starting address (100).

100: i = 0
Step 2: Condition Check

Next, we evaluate the loop condition and store the result in a temporary variable.

101: t1 = i < 5
Step 3: Control Flow

We add conditional jumps based on the condition result.

102: ifFalse t1 goto 106
Step 4: Loop Body

The loop body is executed, and variables are updated.

103: x = x + 1 104: i = i + 1
Step 5: Loop Continuation

Finally, we add a jump back to the condition check.

105: goto 101 106: end
